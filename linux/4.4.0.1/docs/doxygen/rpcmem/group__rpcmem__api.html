void* rpcmem_alloc ( int  heapid,
uint32  flags,
int  size 
)

Allocate a buffer via ION and register it with the FastRPC framework.

Parameters
[in]heapidHeap ID to use for memory allocation.
[in]flagsION flags to use for memory allocation.
[in]sizeBuffer size to allocate.
Returns
Pointer to the buffer on success; NULL on failure.

Examples:

  • Default memory attributes, 2 KB
    1 rpcmem_alloc(RPCMEM_HEAP_ID_SYSTEM, RPCMEM_DEFAULT_FLAGS, 2048);
    Or
    1 rpcmem_alloc_def(2048);
  • Heap 22, uncached, 1 KB
    1 rpcmem_alloc(22, 0, 1024);
    Or
    1 rpcmem_alloc(22, RPCMEM_FLAG_UNCACHED, 1024);
  • Heap 21, cached, 2 KB
    1 rpcmem_alloc(21, RPCMEM_FLAG_CACHED, 2048);
    Or
    1 #include <ion.h>
    2 rpcmem_alloc(21, ION_FLAG_CACHED, 2048);
  • Default memory attributes but from heap 18, 4 KB
    1 rpcmem_alloc(18, RPCMEM_DEFAULT_FLAGS, 4096);

void rpcmem_deinit ( void  )

Deinitialize the RPCMEM Library.

Only call this function once when the RPCMEM Library is no longer required.

This API is mandatory on pre-Lahaina targets IF the client has linked to the rpcmem.a static library. If the client has only linked libadsprpc.so, libcdsprpc.so, or libsdsprpc.so, then the rpcmem_deinit call is not required on any target.

NOTE: This function is not thread safe.

void rpcmem_init ( void  )

Initialize the RPCMEM Library.

Only call this function once before using the RPCMEM Library.

This API is mandatory on pre-Lahaina targets IF the client has linked to the rpcmem.a static library. If the client has only linked libadsprpc.so, libcdsprpc.so, or libsdsprpc.so, then the rpcmem_init call is not required on any target and other rpcmem APIs such as rpcmem_alloc can be called directly.

NOTE: This function is not thread safe.
