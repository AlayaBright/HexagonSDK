Adaptive QoS, DSP driver predicts completion time of a method and send CPU wake up signal to reduce wake up latency Recommended for moderate latency sensitive use cases. It is more power efficient compared to pm_qos control. Poll QoS, After sending invocation to DSP, CPU will enter polling mode instead of waiting for a glink response.
